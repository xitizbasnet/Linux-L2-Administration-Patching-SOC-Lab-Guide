## Overview

This lab provides standardized procedures for preparing, monitoring, and validating server backup operations. It covers pre-backup verification, backup job monitoring, post-backup integrity validation, and coordination with backup administrators to ensure successful data protection and compliance with organizational backup policies.

---

# 🎯 Objective

Perform structured backup administration activities to:

* Verify backup readiness before scheduled jobs
* Monitor backup execution in real time
* Validate backup completion and integrity
* Coordinate with backup teams during failures
* Support backup compliance and recovery readiness

---

# 📌 Scope

This lab applies to backup operations performed using enterprise backup solutions.

| Category             | Components                                                |
| -------------------- | --------------------------------------------------------- |
| **Backup Platforms** | Veeam, Veritas NetBackup, Commvault                       |
| **Activities**       | Pre-Backup Checks, Job Monitoring, Post-Backup Validation |
| **Recovery**         | Restore Verification                                      |
| **Operations**       | Incident Escalation & Backup Team Coordination            |

# Task 7.1 – Pre-Backup Checks

Complete the following verification steps before the scheduled backup window begins.

---

# 1. Verify Backup Agent Status

Confirm that the required backup agent or service is running on the server.

## Veeam Agent

```bash
systemctl status veeam-agent
```

---

## Veritas NetBackup

```bash
systemctl status netbackup
```

---

## Commvault

```bash
systemctl status commvault-simpana
```

---

## Verify Connectivity to the Backup Server

### Test NetBackup Connectivity

```bash
telnet backup-server.company.com 1556
```

> **Note:** Port **1556** is commonly used by Veritas NetBackup.

---

### Verify Network Connectivity

```bash
ping -c 4 backup-server.company.com
```

Ensure the backup server is reachable before backup jobs begin.

---

# 2. Perform a Disk Space Pre-Check

Verify that the source server has sufficient free disk space for temporary backup files.

```bash
df -h
```

Review disk utilization before initiating backup operations.

> **⚠️ Important**
>
> Never configure backups to use the same storage device as the production data.
>
> Always verify that the backup destination resides on separate storage to reduce the risk of data loss.

---

## Verify Application Processes

Identify applications that may require quiescing before backup execution.

```bash
ps aux | grep -E 'oracle|mysql|postgres'
```

Coordinate with the application team if application consistency is required before backup.

---

# Task 7.2 – Monitor Backup Jobs

Monitor backup execution and respond promptly to failures or warnings.

---

# 3. Verify Backup Job Status

## Veritas NetBackup

Display backup job summary.

```bash
/usr/openv/netbackup/bin/admincmd/bpdbjobs -summary
```

---

Display failed backup jobs.

```bash
/usr/openv/netbackup/bin/admincmd/bpdbjobs -report | grep -i failed
```

---

## Veeam Linux Agent

Display configured backup jobs.

```bash
veeamconfig job list
```

---

Display backup session information.

```bash
veeamconfig session list --jobName "Linux_Backup"
```

---

## Review Generic Backup Logs

```bash
tail -100 /var/log/backup/backup.log | grep -iE 'error|fail|warn'
```

Review recent log entries for:

* Errors
* Failed jobs
* Warning messages

---

# 4. Respond to Backup Failures

If a backup job fails:

1. Raise a **Priority 2 (P2)** or **Priority 3 (P3)** incident in **ServiceNow**.
2. Collect and attach backup job logs to the incident.
3. Notify the Backup Team distribution list with:

   * Job ID
   * Server name
   * Error message
4. Follow the organization's escalation process.

---

## Retry the Backup Job

If the maintenance window is still active and approval has been obtained, retry the failed backup.

```bash
veeamconfig job start --name "Linux_Backup"
```

> **Note**
>
> Retry backup jobs only within the approved maintenance window and according to organizational procedures.

---

# Task 7.3 – Post-Backup Verification

Verify that the backup completed successfully and that the backup data is usable.

---

# 5. Verify Backup Completion & Integrity

Perform the following validation steps after backup completion.

### Backup Dashboard

Confirm the backup job status is **SUCCESS** in the backup management console.

---

### Validate Backup File Size

Compare the backup size against historical values.

A significant decrease in backup size may indicate:

* Incomplete backup
* Missing data
* Backup configuration issues

---

### Verify Backup Details (Veritas)

```bash
# Veritas
bpdbjobs -jobid <JOB_ID> -all_columns
```

---

### Verify Available Restore Points (Veeam)

As part of the monthly validation process, confirm that restore points are available.

```bash
veeamconfig restore point list
```

Perform a sample file restore to verify backup usability.

---

### Update Shift Documentation

Record the backup results in the shift report.

Recommended format:

| Server | Backup Job | Status | Duration | Backup Size |
| ------ | ---------- | ------ | -------- | ----------- |

Maintain accurate records for operational tracking and audit purposes.

# ✅ Operational Best Practices

Follow these recommendations during every backup cycle.

---

## 🚨 Treat Backup Failures as High Priority

A failed backup should be treated as a **Priority 2 (P2)** incident by default due to the potential risk of data loss.

Escalate according to the organization's incident management procedures.

---

## 💾 Verify Backup Storage Capacity

Before the backup window begins:

* Confirm sufficient free space exists on the backup repository.
* Verify storage availability.
* Ensure retention policies are not preventing new backups.

---

## 🔄 Perform Regular Restore Testing

A backup is considered reliable only if it can be successfully restored.

Conduct restore validation exercises at least **quarterly** to verify:

* Backup integrity
* Recovery procedures
* Restore performance

---

## 🤝 Coordinate Backup Windows

Schedule backup operations in coordination with application owners to avoid:

* Peak business hours
* High I/O workloads
* Performance degradation
* Application maintenance conflicts

---

## 📊 Maintain a Backup Compliance Dashboard

Track backup compliance using a centralized dashboard.

Recommended fields include:

| Server | Last Successful Backup | Retention Period | Backup Policy |
| ------ | ---------------------- | ---------------- | ------------- |

Regular reviews help identify missed backups and compliance gaps.

---

## ⛔ Do Not Cancel Active Backup Jobs

Never terminate or cancel an active backup without explicit approval from the Backup Team Lead.

Unexpected cancellation may result in:

* Incomplete backups
* Corrupted backup sets
* Data inconsistency
* Failed recovery operations
