## Overview

This quick reference provides a consolidated list of essential Linux commands used by **L2 / SOC Engineers** for daily operations, troubleshooting, monitoring, and system administration in enterprise environments.

---

# 📌 Command Reference Table

| Category            | Command                                      | Purpose                                                  |
| ------------------- | -------------------------------------------- | -------------------------------------------------------- |
| 🖥️ **System Info** | `hostnamectl` / `uname -r`                   | Displays hostname, OS details, and kernel version        |
| 🧠 **CPU**          | `mpstat -P ALL 2 5`                          | Shows per-CPU utilization statistics                     |
| 🧠 **Memory**       | `free -h` / `vmstat -s`                      | Displays memory and swap usage overview                  |
| 💽 **Disk**         | `df -hT` / `iostat -xz 2 5`                  | Shows disk space usage and I/O performance               |
| ⚙️ **Processes**    | `ps aux --sort=-%cpu \| head -10`            | Displays top CPU-consuming processes                     |
| 📜 **Logs**         | `journalctl -xe` / `dmesg -T \| tail -50`    | System error and kernel logs                             |
| 🔧 **Services**     | `systemctl --failed`                         | Lists all failed system services                         |
| 🌐 **Network**      | `ss -tlnp` / `netstat -tuln`                 | Displays listening ports and active services             |
| 💾 **LVM**          | `pvs` / `vgs` / `lvs`                        | Displays LVM physical, volume group, and logical volumes |
| 📦 **Patching**     | `yum check-update` / `apt list --upgradable` | Shows available OS updates                               |
| 🔐 **Security**     | `getenforce` / `firewall-cmd --list-all`     | Checks SELinux status and firewall rules                 |
| 💾 **Backup**       | `veeamconfig job list`                       | Displays backup job status (Veeam)                       |
| 👤 **Users**        | `who` / `last` / `grep sudo /var/log/secure` | User login and sudo activity audit                       |
| 📊 **Performance**  | `sar -u 1 10` / `iotop -o`                   | CPU history and real-time disk I/O monitoring            |

---

# ⚠️ Operational Guidance

* 🧪 All commands must be tested in a **non-production (lab) environment** before production use.
* 🛠️ Always follow your organization’s **Change Management process** before executing impactful commands.
* 📊 Use monitoring and logs together for accurate root cause analysis.
* 🚫 Avoid running destructive commands without validation and approval.
