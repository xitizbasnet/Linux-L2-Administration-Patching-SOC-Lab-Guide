## Overview

This lab defines standard procedures for coordinating with **OEM hardware vendors (Dell & HP)** and internal resolver teams to troubleshoot and resolve infrastructure issues involving servers, storage, networking, and platform components.

It focuses on hardware health validation, vendor escalation workflows, and structured communication with internal support teams.

---

# 🎯 Objective

Coordinate effectively with:

* OEM vendors (HP / Dell)
* Internal resolver groups
* Data center and infrastructure teams

To resolve:

* Hardware failures
* OS-level platform issues
* Storage and RAID problems
* Server lifecycle incidents
* Critical production outages

---

# 📌 Scope

| Category                | Components                                   |

| **Monitoring Areas**    | RAID, disks, hardware sensors, system logs   |

# Task 10.1 – Hardware Health Monitoring

---

# 1. Check Hardware Health (Dell iDRAC)

Access Dell iDRAC interface:

```text id="idrac1"
https://<iDRAC-IP>
```

> ⚠️ Default credentials (production must be changed):
>
> `root / calvin`

---

## Navigate Hardware Dashboard

* Go to: **Dashboard → Hardware Overview**
* Check for:

  * 🟡 Amber alerts (warning)
  * 🔴 Red alerts (critical failure)

---

## Run iDRAC CLI Commands

### Check LED Status

```bash id="dell1"
racadm getled
```

---

### Check Physical Disk Status

```bash id="dell2"
racadm raid get pdisks -o
```

---

### Get System Information

```bash id="dell3"
racadm getsysinfo
```

---

## Check System Event Log (SEL)

```bash id="dell4"
racadm getsel
```

> Used to identify hardware failures, disk issues, and system alerts.

---

# 2. Check Hardware Health (HP iLO)

Access HP iLO interface:

```text id="ilo1"
https://<iLO-IP>
```

---

## Navigate Health Dashboard

* Go to: **Information → Health Summary**
* Validate all subsystem statuses:

  * CPU
  * Memory
  * Storage
  * Fans / Power Supplies

---

## HP CLI Commands

### Get iLO Configuration

```bash id="hp1"
hponcfg -g
```

---

### View Hardware Logs

```bash id="hp2"
hplog -a
```

---

## RAID & Disk Status (HP Agents Required)

### Check RAID Controller Status

```bash id="hp3"
hpacucli ctrl all show status
```

---

### Check Physical Disks

```bash id="hp4"
hpacucli ctrl slot=0 pd all show
```

# Task 10.2 – OEM Vendor Support Coordination

---

# 3. Raise Dell Support Request

---

## Required Information Collection

Before raising a support ticket, collect:

* Service Tag (via dell.com/support)
* iDRAC logs
* OS logs
* Incident symptoms and timestamps

---

## Create Support Request

```text id="dell5"
https://www.dell.com/support
```

---

## Emergency Support Contact

* 📞 Dell ProSupport: `1800-XXX-XXXX` *(region-specific — maintain in runbook)*

---

## P1 Hardware Failure Rule

> 🚨 For P1 hardware failures:
>
> * Call Dell support directly (do not wait for online ticket processing)
> * Provide:
>
>   * Service Tag
>   * Error codes
>   * SEL logs
>   * Physical server location

---

## Change Management Requirement

Before hardware replacement:

* Obtain **Change Request (CRQ)** approval from Change Manager
* Ensure vendor access is authorized

---

# 4. Internal Resolver Group Coordination

Coordinate issues with internal support teams based on domain responsibility.

---

## Resolver Groups

| Team                | Responsibility                                    |
| ------------------- | ------------------------------------------------- |
| 🗄️ Storage Team    | SAN issues, multipath failures, disk provisioning |
| 🌐 Network Team     | VLANs, switches, NIC bonding, routing issues      |
| 🛢️ DBA Team        | Database errors, performance, connectivity        |
| 🧩 Application Team | Application failures, deployments, runtime issues |
| 🔐 Security Team    | Access violations, antivirus alerts, compliance   |

---

## P1 Incident Coordination

For critical incidents:

* 🔴 Always initiate a **bridge call (Teams/Zoom)**
* Include all resolver groups
* Ensure real-time collaboration until resolution

---

## Post-Incident Communication

After resolution:

* Send a **Bridge Call Summary Email**
* Include:

  * Root cause
  * Actions taken
  * Owners assigned
  * Preventive measures

# ✅ Operational Best Practices

---

## 📚 Maintain Vendor Documentation

* Store OEM support contacts in a shared wiki
* Ensure URLs and escalation contacts are always accessible

---

## 📡 Enable Hardware Alerts

* Configure SMTP alerts in:

  * iDRAC
  * iLO

> Hardware systems often notify issues before OS-level detection.

---

## 🛑 Vendor Access Control

* Always obtain a **Vendor SR number** before granting data center access
* No physical access without documented approval

---

## 📄 Change Management Compliance

* Hardware replacements require a **Change Request (CRQ)**
* Emergency changes still require documentation and post-review

---

## 📊 Maintain Resolver Matrix

Maintain a structured contact matrix including:

* Names
* Emails
* Phone numbers
* Escalation paths

---

## 📝 Log All Vendor Interactions

Every interaction must include:

* Timestamp
* Vendor name
* Reference / SR number
* Action taken

Maintain this in shift logs for audit and traceability.
