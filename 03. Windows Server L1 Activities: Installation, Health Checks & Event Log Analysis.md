## Overview

This lab covers the fundamental **Level 1 (L1)** administrative tasks for Windows Server environments. It includes system health verification, service monitoring, disk management, and Windows Event Log analysis using **PowerShell** and built-in Windows administration tools.

---

Perform **Windows Server L1 administrative tasks** including:

* Server health verification
* Resource monitoring
* Service management
* Disk and volume management
* Windows Event Log analysis
* Basic troubleshooting using PowerShell

# Task 3.1 – Windows Server Health Check

Perform the following health checks before starting maintenance activities or troubleshooting production servers.

---

# 1. Verify System Information & Resource Utilization

Open **PowerShell** with **Administrator** privileges and execute the following commands.

## Display System Information

```powershell
Get-ComputerInfo | Select-Object OsName, OsVersion, CsName, TotalPhysicalMemory
```

Displays:

* Operating System name
* Operating System version
* Computer hostname
* Total physical memory

---

## Identify High CPU Usage Processes

```powershell
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10
```

Review the top CPU-consuming processes to identify abnormal resource utilization.

---

## Check Available Disk Space (C: Drive)

```powershell
Get-PSDrive C | Select-Object Used, Free
```

Verify the used and available storage capacity of the system drive.

---

# 2. Monitor Windows Services

Review the status of Windows services that are configured to start automatically.

## Identify Stopped Automatic Services

```powershell
Get-Service | Where-Object {$_.Status -eq 'Stopped' -and $_.StartType -eq 'Automatic'}
```

Investigate any services that are unexpectedly stopped.

---

## Restart a Windows Service

Example:

```powershell
Restart-Service -Name 'Spooler' -Force
```

Restart services **only after verifying that it complies with organizational change management procedures.**

---

## Review Recent Application Errors

```powershell
Get-EventLog -LogName Application -EntryType Error -Newest 20
```

Check the latest application errors for service-related failures or application issues.

---

# 3. Disk Management

Monitor storage utilization and verify disk health.

## Display File System Drives

```powershell
Get-PSDrive -PSProvider FileSystem
```

---

## Display Physical Disk Information

```powershell
Get-Disk | Format-Table -AutoSize
```

---

## Display Volume Information

```powershell
Get-Volume | Format-Table DriveLetter, FileSystem, Size, SizeRemaining -AutoSize
```

Review:

* Drive letter
* File system type
* Total disk size
* Available free space

---

## Disk Utilization Thresholds

| Disk Usage    | Required Action                                                       |
| ------------- | --------------------------------------------------------------------- |
| **Above 85%** | ⚠️ Generate an alert and investigate storage growth.                  |
| **Above 90%** | 🚨 Escalate immediately according to the incident management process. |

---

## Extend a Disk Partition

If unallocated space is available, extend the partition using:

```powershell
Resize-Partition -DriveLetter C -Size (Get-PartitionSupportedSize -DriveLetter C).SizeMax
```

> **⚠️ Important**
>
> Verify that a valid change request has been approved before modifying production disk partitions.

# 4. Windows Event Log Analysis

Windows Event Logs provide critical information for troubleshooting system, application, and security-related issues.

---

## Open Event Viewer

Launch Event Viewer using either:

### Run Dialog

```text
eventvwr.msc
```

### PowerShell

```powershell
Get-EventLog -LogName System -EntryType Error,Warning -Newest 50 | Format-Table -AutoSize
```

Review recent **System** warnings and errors.

---

## Review Failed Login Attempts

```powershell
Get-EventLog -LogName Security -InstanceId 4625 -Newest 20
```

Monitor failed authentication attempts to identify potential security issues.

---

## Review Critical System Events

```powershell
Get-WinEvent -FilterHashtable @{LogName='System'; Level=2} -MaxEvents 30
```

Displays recent **Critical** system events requiring investigation.

---

## Export Event Logs

Export system event logs for troubleshooting or audit purposes.

```powershell
Get-EventLog -LogName System -Newest 200 | Export-Csv C:\Logs\syslog.csv -NoTypeInformation
```

Store exported logs securely according to your organization's retention policy.

# ✅ Operational Best Practices

Follow these best practices during routine Windows Server administration.

---

## 🔐 Run PowerShell with Administrative Privileges

Always launch **PowerShell as Administrator** when performing:

* Service management
* Disk management
* System configuration
* Administrative troubleshooting

---

## 📋 Monitor Critical Event IDs

Filter Event Viewer for the following commonly used Event IDs:

| Event ID | Description                   |
| -------- | ----------------------------- |
| **7045** | New Windows service installed |
| **4625** | Failed user login attempt     |
| **41**   | Unexpected system reboot      |
| **1074** | Planned shutdown or restart   |
| **6008** | Unexpected system shutdown    |

These events should be reviewed regularly during health checks and incident investigations.

---

## 📊 Maintain a Baseline Health Report

Create a standard **Windows Server Health Report** (Excel or approved reporting tool).

Record the following information at the beginning of every shift:

* CPU utilization
* Memory usage
* Disk usage
* Running services
* Event log summary
* Outstanding alerts

Maintaining historical health reports helps identify performance trends and recurring issues.

---

## 🖥️ Collect Hardware Inventory

Use the following tools to gather hardware inventory information when required:

* `wmic`
* `Get-WmiObject`

These commands can provide details about:

* CPU
* Memory
* Disk
* BIOS
* Manufacturer
* System Model

---

## ⚠️ Disk Management Precautions

Never perform the following operations on a production server without:

* A verified backup
* An approved change request
* Appropriate maintenance window authorization

Restricted operations include:

* Formatting partitions
* Shrinking partitions
* Extending production volumes
* Modifying storage layouts
